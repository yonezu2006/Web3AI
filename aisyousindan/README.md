# 相性診断ゲーム

出店イベント向け、2台のMacで遊ぶ相性診断Webアプリ。ニックネーム入力→顔写真撮影→8問クイズに答えると、AIが相性%・コメント・おすすめ旅行スポットを診断し、顔ハメ合成画像とQRコード付きの結果を生成します。

## 技術構成

- サーバー: Node.js + Express + Socket.io(ホストMac1台で起動)
- クライアント: ブラウザ(HTML/JS、`getUserMedia`でカメラ撮影)
- 顔の切り抜き: 撮影画面のガイド枠(点線の楕円)の範囲をそのまま切り抜きます(外部ライブラリ不要)。
- 診断ロジック: OpenAI API(`.env`の`OPENAI_API_KEY`がダミー/未設定の場合はローカルのフォールバックロジックで診断します。イベント当日に実キーがなくても動作します)
- 画像合成: ブラウザのcanvas(外部API不要、完全ローカル処理)
- PDF生成: jsPDF(`public/vendor/jspdf.umd.min.js` にローカル同梱)

## 事前準備(初回のみ・インターネット接続が必要)

```bash
cd aisyousindan
npm install
```

`.env` を編集し、OpenAIの実APIキーを使う場合は設定してください(ダミーのままでもフォールバック診断で動作します)。

```
PORT=3000
OPENAI_API_KEY=sk-xxxxxxxx
OPENAI_MODEL=gpt-4o-mini
PUBLIC_PORT=3001
NGROK_AUTHTOKEN=
```

`NGROK_AUTHTOKEN` を設定すると、結果ページのQRコードが**ホストのWi-Fi/テザリングに繋がっていない普通のスマホ(モバイルデータ)からも開けるURL**になります。[ngrokダッシュボード](https://dashboard.ngrok.com/get-started/your-authtoken)で無料登録・取得できます。未設定の場合はLAN内(同じWi-Fi/テザリング)からのみ開けるURLになります。2台のMac同士のゲーム本体の通信は、この設定に関わらず引き続きLAN内で行われます。

## 起動方法(イベント当日・ホストMac側)

```bash
cd aisyousindan
npm start
```

起動すると以下のようなログが出ます。

```
相性診断ゲーム サーバー起動: https://localhost:3000
同じWi-Fi内の別のMacからは https://<ホストのIPアドレス>:3000 でアクセスしてください
```

- **ホストMac自身**: ブラウザで `https://localhost:3000` を開く
- **もう1台のMac**: 表示された `https://<ホストのIPアドレス>:3000` にブラウザでアクセスする(同じWi-Fi、またはホストのテザリングに接続していること)

IPアドレスが分からなくなった場合は、ホストMacのターミナルで以下でも確認できます。

```bash
ipconfig getifaddr en0
```

または起動時のログ、もしくは `https://localhost:3000/api/server-info` にアクセスすると `{"ip":"...","port":3000,"protocol":"https"}` が返ります。

### HTTPS(自己署名証明書)について

カメラ(`getUserMedia`)はブラウザの仕様上、`localhost`かHTTPSでしかアクセスできません。もう1台のMacからはIPアドレスでアクセスするため、サーバーは起動時に自己署名証明書を自動生成してHTTPSで待ち受けます(`certs/`フォルダに保存、gitでは管理していません)。

初回アクセス時にブラウザで「この接続ではプライバシーが保護されません」等の警告が出ますが、**「詳細設定」→「(IPアドレス)にアクセスする(安全ではありません)」を選べば正常に利用できます**。これは自己署名証明書のための警告であり、この場だけの一時的なローカルサーバーなので問題ありません。

ホストMacのIPアドレスが変わった場合は、サーバー再起動時に証明書が自動的に作り直されます。

## 遊び方の流れ

1. 2台とも「ニックネーム入力」画面が表示される(最初に接続した方がプレイヤーA、次がプレイヤーB)
2. 各自ニックネームを入力→顔写真を撮影→8問の二択クイズに回答(相手の回答は見えません)
3. 両方の回答が揃うと自動で「診断中...」→「診断結果」画面に遷移
4. 結果画面下部の「二人に合うおすすめの旅行スポットは」ボタンは各自が好きなタイミングで押せる(押した側だけがその画面で旅行スポット画面に遷移する)。顔ハメ合成画像とQRコードは、先に押した側が保存し、後から押した側にも同じ結果(同じQR)が表示される
5. スマホでQRコードを読み取ると、結果画像を表示するページが開く(長押しで画像を保存可能)
6. 次のペアに交代する際は、旅行スポット画面の「終了して次のペアへ」ボタンを押すと両方の画面がニックネーム入力に戻る

3台目以降が接続した場合は「ただいま使用中です」と表示され、前のペアが終わるまで待機になります(同時に遊べるのは1組のみ)。

## 背景カテゴリの追加方法

このアプリは、背景写真に**実際に写っている人物の顔**に直接プレイヤーの顔写真をはめ込む方式です。写真ごとに人物の位置が違うため、`data/categories.json` にカテゴリを追加する際は、背景画像を `hukei/` フォルダに置いた上で、そのカテゴリ専用の `canvas`(画像のピクセルサイズ)と `faceSlots`(写っている人物ごとの顔位置)を指定します。

```json
{
  "categories": {
    "your-new-category": {
      "label": "表示名",
      "backgroundImage": "hukei/ファイル名.jpg",
      "description": "説明文",
      "keywords": ["キーワード1", "キーワード2"],
      "spots": ["地名1", "地名2"],
      "canvas": { "width": 画像の幅(px), "height": 画像の高さ(px) },
      "faceSlots": [
        { "gender": "male | female | null", "faceHole": { "x": 左上x, "y": 左上y, "width": 幅, "height": 高さ }, "rotation": 度数(省略可), "feather": ぼかし幅px(省略可) },
        { "gender": "male | female | null", "faceHole": { "x": 左上x, "y": 左上y, "width": 幅, "height": 高さ }, "rotation": 度数(省略可), "feather": ぼかし幅px(省略可) }
      ]
    }
  }
}
```

- `canvas` は背景画像そのもののピクセルサイズです。`file 画像パス` や Finderの「情報を見る」で確認できます。このサイズを画像の実サイズと一致させておくと、`faceHole` の座標を元画像のピクセル座標そのままで指定でき、ズレの原因になる拡大縮小の計算が不要になります。
- `faceSlots` は写真に写っている人物の人数分(通常2人)だけ配列に並べます。各人物の顔(宇宙服ならバイザー越しの顔)を囲むだいたいの矩形を、左上を原点として `x, y, width, height` で指定してください。
- `rotation`(省略可、度数): 人物の頭が傾いている場合、その角度だけ顔ハメの楕円ごと回転させます。省略時は0(回転なし)。
- `feather`(省略可、px): 顔ハメの縁をこのピクセル数だけぼかして背景になじませます。省略時は縁がくっきりした通常の楕円クリップになります。
- `gender` は「その人物が男性/女性どちらに見えるか」のヒントです。2つのスロットの性別指定が異なる場合(例:左が男性・右が女性)、プレイヤーの自己申告の性別と自動的に対応付けられます。どちらの性別でも違和感がない場合(服装等で性別を問わない場合)は `null` にしてください。
- 座標のおおまかな調べ方: 画像をPreview.appなどで開いて顔のだいたいの位置を目視で読み取るか、簡単なHTMLに `<img>` で画像を表示し、クリック位置を `event.offsetX`/`event.offsetY` で `console.log` する程度の簡易ツールでも十分です。ぴったり合わなくても構いません。実装後に実際に合成画面を1回描画し、元の写真と見比べながら数ピクセル単位で微調整するのが確実です。

現在は `hukei/` フォルダの3枚の画像(`洋風の町並み.jpg` / `宇宙.jpg` / `花園.jpg`)を使い、「洋風の町並み」「宇宙」「花園」の3カテゴリで動作する構成になっています。

## 保存されるデータ

- 診断終了後、PDFと結果画像は `kekka/<ニックネーム1>_<ニックネーム2>/` フォルダに保存されます。
- 保存が完了すると、撮影した顔写真の一時ファイル(`tmp_faces/`)はプライバシー保護のため自動的に削除されます。
- QRコードのリンク先(`/kekka/<固有ID>`)はペアごとにランダムな固有IDになっており、他のペアの結果にはアクセスできません。

## 注意事項

- **イベント終了後は、必ずサーバー(`npm start`のプロセス)を完全に停止してください。** サーバーを起動したままにすると、発行済みのQRコードURLから引き続き結果ページにアクセスできてしまいます。サーバーを停止すればアクセスできなくなります。
- `kekka/` フォルダには参加者の顔写真・PDFが保存されるため、`.gitignore`でgit管理対象から除外しています。イベント終了後は必要に応じてフォルダの中身を削除してください。
